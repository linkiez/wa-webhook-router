# Deployment Guide for On-Premise Server

## Prerequisites

- Docker and Docker Compose installed
- AWS account with access to SQS queue
- Queue URL from `wa-webhook-lambda-to-queue` deployment

## Step-by-Step Deployment

### 1. Create AWS IAM Credentials

Run the credential creation script:

```bash
./create-sqs-credentials.sh
```

This creates:
- IAM user: `sqs-consumer-user` (or custom name via `IAM_USER_NAME` env var)
- SQS read-only policy (ReceiveMessage, DeleteMessage, GetQueueAttributes, GetQueueUrl)
- Access credentials saved to `.env.credentials`

### 2. Configure Environment

```bash
# Copy generated credentials
cp .env.credentials .env

# Edit and add routing configuration
nano .env
```

Required variables in `.env`:
```env
# AWS Credentials (auto-generated)
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=...
AWS_REGION=us-east-1
QUEUE_URL=https://sqs.us-east-1.amazonaws.com/...

# Add your routing configuration
DESTINATION_HOST=http://your-backend:3000
PHONE_ROUTES=5511999999999::/webhooks/meta::token|5511888888888::/webhooks/whatsapp
```

### 3. Build Docker Image

```bash
docker build -t wa-webhook-router:latest .
```

### 4. Deploy with Docker Compose

```bash
# Start service
docker-compose up -d

# Check logs
docker-compose logs -f wa-webhook-router

# Check status
docker-compose ps
```

### 5. Verify Deployment

Check logs for successful startup:
```bash
docker-compose logs wa-webhook-router | grep "Polling queue"
```

Expected output:
```
[SQS Consumer] Starting...
[SQS Consumer] Queue URL: https://sqs.us-east-1.amazonaws.com/...
[SQS Consumer] AWS Region: us-east-1
[SQS Consumer] Polling queue: ...
[SQS Consumer] Available routes: ['5511999999999', '5511888888888']
```

## Environment Variables Reference

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `AWS_ACCESS_KEY_ID` | Yes | AWS IAM access key | `AKIA...` |
| `AWS_SECRET_ACCESS_KEY` | Yes | AWS IAM secret key | Generated by script |
| `AWS_REGION` | Yes | AWS region | `us-east-1` |
| `QUEUE_URL` | Yes | SQS queue URL | `https://sqs.us-east-1...` |
| `DESTINATION_HOST` | Yes | Base URL for routing | `http://app:3000` |
| `PHONE_ROUTES` | Yes | Phone to endpoint mapping | `phone::/path::token\|...` |

## Phone Routes Format

```
PHONE_ROUTES=phone1::path1::token1|phone2::path2|phone3::https://full-url.com/path
```

- `phone`: Phone number from Meta webhook
- `path`: Endpoint path (appended to DESTINATION_HOST) or full URL
- `token`: Optional Bearer token for Authorization header
- `|`: Separator between routes

## Troubleshooting

### No Messages Received
1. Check queue has messages: `aws sqs get-queue-attributes --queue-url <URL> --attribute-names ApproximateNumberOfMessages`
2. Verify credentials: `aws sqs list-queues` (should work with created credentials)
3. Check network connectivity to AWS

### Authentication Errors
```bash
# Test credentials
docker exec wa-webhook-router sh -c 'env | grep AWS'

# Recreate credentials
./create-sqs-credentials.sh
```

### Routing Errors
- Check `PHONE_ROUTES` format in `.env`
- Verify phone numbers match those in Meta webhooks
- Check `DESTINATION_HOST` is accessible from container

## Security Best Practices

1. **Minimal IAM Permissions**: Script creates read-only SQS access
2. **Credential Rotation**: Rotate credentials periodically:
   ```bash
   ./create-sqs-credentials.sh  # Recreates credentials
   docker-compose restart
   ```
3. **Network Isolation**: Use Docker networks to isolate services
4. **Secrets Management**: Consider using Docker secrets for production:
   ```yaml
   services:
     wa-webhook-router:
       secrets:
         - aws_access_key_id
         - aws_secret_access_key
   secrets:
     aws_access_key_id:
       file: ./secrets/aws_access_key_id.txt
   ```

## Production Recommendations

1. **Use AWS IAM Roles** (if running on AWS EC2/ECS):
   - Attach IAM role to instance/task
   - Remove `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` from `.env`

2. **Health Checks**: Configure monitoring for container health

3. **Auto-restart**: Already configured in `docker-compose.yml` as `restart: unless-stopped`

4. **Resource Limits**: Adjust in `docker-compose.yml` based on load

5. **Logging**: Configure log aggregation (ELK, Splunk, CloudWatch)
